| # | Vị trí                                            | Loại vấn đề                   | Mô tả hậu quả                                                                                                                                       |
| - | ------------------------------------------------- | ----------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1 | `<%! private int requestCounter = 0; %>`          | Race Condition / Sai thiết kế | Biến khai báo bằng Declaration trở thành biến dùng chung trong servlet JSP, không an toàn khi nhiều request truy cập đồng thời, dễ cho kết quả sai. |
| 2 | Khối `<% ... %>` đầu trang                        | Scriptlet                     | JSP chứa Java code trực tiếp, làm View xử lý cả dữ liệu và logic, vi phạm nguyên tắc Thin View, khó đọc và khó bảo trì.                             |
| 3 | Vòng lặp `for` và khối `if/else` trong JSP        | Logic trong View              | Logic xếp loại bị đặt trong giao diện, làm JSP quá tải trách nhiệm, khó kiểm thử và khó tái sử dụng.                                                |
| 4 | `<td><%= sv.getFullName() %></td>`                | XSS                           | In dữ liệu trực tiếp bằng Expression không escape HTML, có nguy cơ chèn mã độc nếu dữ liệu đầu vào không an toàn.                                   |
| 5 | `<td><%= sv.getScore()%>;</td>`                   | Lỗi cú pháp hiển thị          | Dấu `;` bị in ra ngoài giao diện, làm sai dữ liệu trình bày.                                                                                        |
| 6 | `<!-- Tiêu đề trang báo cáo -->`                  | Sai loại comment              | HTML comment vẫn xuất hiện trong mã nguồn trả về trình duyệt, không phù hợp cho ghi chú nội bộ.                                                     |
| 7 | `import="java.util.List, com.demo.model.Student"` | Phụ thuộc Java trong JSP      | JSP phải biết trực tiếp kiểu dữ liệu Java, làm tăng mức phụ thuộc giữa View và Model.                                                               |
| 8 | `<%= title %>`, `<%= rank %>`, `<%= i + 1 %>`     | Expression cũ                 | Dùng cú pháp JSP Expression thay vì EL làm code dài, cũ, khó bảo trì và không thống nhất với chuẩn JSP hiện đại.                                    |
